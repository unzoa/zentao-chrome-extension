# 禅道 Chrome 扩展安装说明

## 功能特性

- 自动识别禅道页面，在右上角显示"任务抽屉"按钮
- 右键点击任务行，选择"添加到抽屉"快速添加任务
- 支持手动添加任务
- 拖拽调整任务排序
- 编辑任务分类（CCMS、BDS、VISITRACK、HIM、SPAS）
- 编辑任务状态（挂起、进行中、已完成）
- 编辑发布环境（test、uat、prod）
- 数据本地存储，刷新页面不丢失

## 安装步骤

### 1. 准备图标文件

由于 Chrome 扩展需要图标文件，请按以下步骤生成图标：

#### 方法一：使用在线工具
1. 访问 https://www.favicon-generator.org/ 或类似网站
2. 上传任意图片或使用默认图标
3. 生成并下载图标包
4. 将下载的图标重命名为：
   - icon16.png (16x16)
   - icon48.png (48x48)
   - icon128.png (128x128)
5. 将这些文件放到项目根目录

#### 方法二：使用项目提供的生成器
1. 在浏览器中打开 `generate-icons.html`
2. 页面会自动下载三个图标文件
3. 将下载的图标文件放到项目根目录

### 2. 加载扩展到 Chrome

1. 打开 Chrome 浏览器
2. 在地址栏输入 `chrome://extensions/`
3. 打开右上角的"开发者模式"开关
4. 点击"加载已解压的扩展程序"
5. 选择项目文件夹 `/Users/unzoa/github/zentao-chrome-extension`
6. 扩展加载成功！

### 3. 使用扩展

1. 访问禅道网站
2. 页面右上角会出现"📋 任务抽屉"按钮
3. 点击按钮打开任务抽屉
4. 在禅道任务列表中，右键点击任务行
5. 选择"添加到抽屉"
6. 任务会自动添加到抽屉中
7. 在抽屉中可以：
   - 拖拽调整任务顺序
   - 选择任务分类
   - 设置任务状态
   - 选择发布环境
   - 删除任务

## 数据结构

每个任务包含以下字段：

```json
{
  "id": "任务ID",
  "project": "项目名称",
  "user": "负责人",
  "sort": "排序序号",
  "system": "分类 (CCMS/BDS/VISITRACK/HIM/SPAS)",
  "status": "状态 (挂起/进行中/已完成)",
  "env": "环境 (test/uat/prod)"
}
```

## 数据存储

- **存储方式**: 使用浏览器 localStorage 存储
- **存储键**: `zentaoTasks`
- **数据持久化**: 数据保存在浏览器本地，关闭浏览器不会丢失
- **唯一性检查**: 根据任务 ID 判断唯一性，重复 ID 会提示"任务ID已存在，不能重复添加"
- **数据备份**: 可以通过浏览器开发者工具查看和导出 localStorage 数据

## 技术栈

- Chrome Extension Manifest V3
- Vanilla JavaScript
- Chrome Storage API
- Context Menus API

## 注意事项

- 扩展需要识别禅道页面的 URL，确保访问的 URL 包含 "zentao" 字符串
- 任务数据存储在本地 Chrome Storage 中，清除浏览器数据会丢失任务
- 扩展在所有页面都会加载，但只在禅道页面显示抽屉按钮

## 开发调试

1. 修改代码后，在 `chrome://extensions/` 页面点击扩展的刷新按钮
2. 打开禅道页面测试功能
3. 使用 Chrome DevTools 调试 content script 和 background script

## 常见问题

**Q: 为什么看不到抽屉按钮？**
A: 确保访问的 URL 包含 "zentao" 字符串，扩展只在禅道页面显示按钮。

**Q: 右键菜单没有"添加到抽屉"选项？**
A: 确保选中了任务行的文本内容，右键菜单只在有选中文本时显示。

**Q: 任务数据丢失了？**
A: 数据存储在本地 Chrome Storage 中，清除浏览器数据或卸载扩展会导致数据丢失。

**Q: 如何备份任务数据？**
A: 目前版本不支持导出，建议定期截图或手动记录重要任务信息。
